Resumen General

La API debe proveer endpoints tanto para dueños de fincas como para los usuarios que desean arrendar, es asi que se necesitan endpoints que en la mayoria de casos seran unicos para uno de los 2 roles, mientras que otros podran servir para ambos roles, en estos endpoints es fundamental el uso de la autorizacion para identificar correctamente la logica que se debe emplear dependiendo del tipo de usuario

En total se tienen 22 endpoints, de los cuales 9 endpoints cumplen historias de usuario de LANDLORD y USER al mismo tiempo, es decir, endpoints como crear cuenta, o agregar ratings, son 1 para ambos roles. Por otro lado se tienen 7 endpoints solo para LANDLORDS, y finalmente 6 para USER.
